# 클로드 단위업무: 공지사항 게시판 (UW-00001)

## 1. 개요

| 항목 | 내용 |
|------|------|
| 단위업무ID | UW-00001 |
| 단위업무명 | 공지사항 게시판 |

#### 기능: [FN-00001] 게시판 목록 조회
| 항목 | 내용 |
|------|------|
| 기능ID | FN-00001 |
| 기능명 | 게시판 목록 조회 |
| 기능유형 | SELECT |
| API | `GET /api/board` |

**Input**
| 파라미터 | 타입 | 필수 | DB 매핑 | 설명 |
|----------|------|------|---------|------|
| projectId | number | Y (세션) | project_id | 프로젝트 식별자 |
| boardTypeCd | string | N | board_type_cd | 유형 필터 (null이면 전체) |
| keyword | string | N | board_title_nm | 제목 LIKE 검색 |
| startDt | string | N | reg_dt | 기간 시작 (>=) |
| endDt | string | N | reg_dt | 기간 종료 (<=) |
| page | number | Y | - | 페이지 번호 (1부터) |
| size | number | Y | - | 페이지당 건수 (기본 20) |

**Output**
| 필드 | 타입 | DB 매핑 | 설명 |
|------|------|---------|------|
| boardId | number | board_id | 게시글 ID |
| boardTypeCd | string | board_type_cd | 유형코드 |
| boardTitleNm | string | board_title_nm | 제목 |
| regUserId | string | reg_user_id | 작성자 ID |
| regUserNm | string | (tb_cm_user JOIN) | 작성자명 |
| regDt | string | reg_dt | 등록일시 |
| viewCnt | number | view_cnt | 조회수 |
| fixYn | string | fix_yn | 상단고정 여부 |
| attachYn | string | (서브쿼리) | 첨부파일 존재 여부 Y/N |
| totalCount | number | COUNT(*) | 전체 건수 |

**처리 로직**
```
1. project_id = 세션에서 획득
2. del_yn = 'N' 필터
3. 검색 조건 적용:
   - boardTypeCd IS NOT NULL → board_type_cd = :boardTypeCd
   - keyword IS NOT NULL → board_title_nm LIKE '%' || :keyword || '%'
   - startDt IS NOT NULL → reg_dt >= :startDt::timestamp
   - endDt IS NOT NULL → reg_dt <= :endDt::timestamp + interval '1 day'
4. 정렬: fix_yn DESC, reg_dt DESC (상단고정 먼저, 그 다음 최신순)
5. 페이징: LIMIT :size OFFSET (:page - 1) * :size
6. 첨부파일 존재 여부: tb_cm_attach_file에서 ref_type_cd = 'BOARD' AND ref_id = board_id 존재 확인
```

**SQL 개요**
```sql
SELECT 
    b.board_id,
    b.board_type_cd,
    b.board_title_nm,
    b.reg_user_id,
    b.reg_dt,
    b.view_cnt,
    b.fix_yn,
    CASE WHEN EXISTS (
        SELECT 1 FROM tb_cm_attach_file f 
        WHERE f.ref_type_cd = 'BOARD' AND f.ref_id = b.board_id AND f.del_yn = 'N'
    ) THEN 'Y' ELSE 'N' END AS attach_yn,
    COUNT(*) OVER() AS total_count
FROM tb_cm_board b
WHERE b.project_id = :projectId
  AND b.del_yn = 'N'
  AND (:boardTypeCd IS NULL OR b.board_type_cd = :boardTypeCd)
  AND (:keyword IS NULL OR b.board_title_nm LIKE '%' || :keyword || '%')
  AND (:startDt IS NULL OR b.reg_dt >= :startDt::timestamp)
  AND (:endDt IS NULL OR b.reg_dt <= :endDt::timestamp + interval '1 day')
ORDER BY b.fix_yn DESC, b.reg_dt DESC
LIMIT :size OFFSET :offset
```

**업무 규칙**
- 검색 결과 0건이면 "등록된 게시글이 없습니다" 안내 메시지 표시
- 상단고정(fix_yn='Y') 게시글은 페이지와 무관하게 항상 최상단 표시
- 기간 검색 시 종료일은 해당일 23:59:59까지 포함

```
3. 조회수 증가: UPDATE tb_cm_board SET view_cnt = view_cnt + 1 WHERE board_id = :boardId
   (본인 글 조회 시에도 증가 — 요구사항에 따라 변경 가능)
4. 첨부파일 조회: ref_type_cd = 'BOARD', ref_id = board_id, del_yn = 'N'
5. isOwner: 세션 user_id == reg_user_id 비교
```

**SQL 개요**
```sql
-- 조회수 증가
UPDATE tb_cm_board SET view_cnt = view_cnt + 1 WHERE board_id = :boardId AND del_yn = 'N';

-- 상세 조회
SELECT b.*, 
FROM tb_cm_board b
WHERE b.board_id = :boardId AND b.del_yn = 'N';

-- 첨부파일
SELECT file_id, file_origin_nm, file_size, file_ext_nm
FROM tb_cm_attach_file
WHERE ref_type_cd = 'BOARD' AND ref_id = :boardId AND del_yn = 'N'
ORDER BY reg_dt ASC;
```

#### 기능: [FN-00003] 게시판 삭제
| 항목 | 내용 |
|------|------|
| 기능ID | FN-00003 |
| 기능명 | 게시판 삭제 |
| 기능유형 | DELETE |
| API | `DELETE /api/board/{boardId}` |

**Input**
| 파라미터 | 타입 | 필수 | DB 매핑 | 설명 |
|----------|------|------|---------|------|
| boardId | number | Y (경로) | board_id | 게시글 식별자 |

**Output**
| 필드 | 타입 | 설명 |
|------|------|------|
| success | boolean | 성공 여부 |
| message | string | 결과 메시지 |

**처리 로직**
```
1. 게시글 존재 확인 (del_yn = 'N')
2. 권한 확인: 세션 user_id == reg_user_id (본인 글만 삭제 가능)
3. 권한 없으면 403 에러
4. 논리삭제: del_yn = 'Y', mod_user_id, mod_dt 업데이트
5. 첨부파일도 논리삭제: ref_type_cd = 'BOARD', ref_id = board_id → del_yn = 'Y'
6. 삭제 후 목록 화면으로 리다이렉트
```

**SQL 개요**
```sql
-- 게시글 논리삭제
UPDATE tb_cm_board 
SET del_yn = 'Y', mod_user_id = :userId, mod_dt = NOW()
WHERE board_id = :boardId AND del_yn = 'N';

-- 첨부파일 논리삭제
UPDATE tb_cm_attach_file 
SET del_yn = 'Y'
WHERE ref_type_cd = 'BOARD' AND ref_id = :boardId AND del_yn = 'N';
```

**업무 규칙**
- 삭제 전 확인 다이얼로그: "게시글을 삭제하시겠습니까? 첨부파일도 함께 삭제됩니다."
- 삭제 후 "삭제되었습니다" 토스트 메시지 + 목록 화면 이동

#### 기능: [FN-00004] 게시판 등록
| 항목 | 내용 |
|------|------|
| 기능ID | FN-00004 |
| 기능명 | 게시판 등록 |
| 기능유형 | INSERT |
| API | `POST /api/board` |

**Input**
| 파라미터 | 타입 | 필수 | DB 매핑 | 검증 | 설명 |
|----------|------|------|---------|------|------|
| boardTitleNm | string | Y | board_title_nm | 1~300자, 빈값 불가 | 제목 |
| boardTypeCd | string | Y | board_type_cd | NORMAL 또는 NOTICE | 유형 |
| boardCn | string | Y | board_cn | 빈값 불가 | 마크다운 본문 |
| fixYn | string | N | fix_yn | Y 또는 N | 기본값 'N' |
| files | File[] | N | tb_cm_attach_file | 파일당 10MB, 최대 5개 | multipart |

**Output**
| 필드 | 타입 | 설명 |
|------|------|------|
| boardId | number | 생성된 게시글 ID |
| message | string | "등록되었습니다" |

**처리 로직**
```
1. 프론트 검증: 제목 빈값, 본문 빈값, 파일 크기/개수
2. 서버 검증: 동일 검증 반복 (프론트 우회 방지)
3. project_id = 세션에서 획득
4. tb_cm_board INSERT
5. 첨부파일 있으면:
   a. 파일을 스토리지에 저장 (file_stored_nm = UUID + 확장자)
   b. tb_cm_attach_file INSERT (ref_type_cd = 'BOARD', ref_id = 생성된 board_id)
6. 성공 응답 + 목록 화면 이동
```

**SQL 개요**
```sql
-- 게시글 등록
INSERT INTO tb_cm_board (
    project_id, board_title_nm, board_cn, board_type_cd, 
    fix_yn, view_cnt, del_yn, reg_user_id, reg_dt
) VALUES (
    :projectId, :boardTitleNm, :boardCn, :boardTypeCd,
    :fixYn, 0, 'N', :userId, NOW()
) RETURNING board_id;

-- 첨부파일 등록 (파일별 반복)
INSERT INTO tb_cm_attach_file (
    ref_type_cd, ref_id, file_origin_nm, file_stored_nm, 
    file_path, file_size, file_ext_nm, del_yn, reg_user_id, reg_dt
) VALUES (
    'BOARD', :boardId, :fileOriginNm, :fileStoredNm,
    :filePath, :fileSize, :fileExtNm, 'N', :userId, NOW()
);
```

#### 기능: [FN-00005] 게시판 수정
| 항목 | 내용 |
|------|------|
| 기능ID | FN-00005 |
| 기능명 | 게시판 수정 |
| 기능유형 | UPDATE |
| API | `PUT /api/board/{boardId}` |

**Input**
| 파라미터 | 타입 | 필수 | DB 매핑 | 검증 | 설명 |
|----------|------|------|---------|------|------|
| boardId | number | Y (경로) | board_id | 존재 확인 | |
| boardTitleNm | string | Y | board_title_nm | 1~300자 | |
| boardTypeCd | string | Y | board_type_cd | NORMAL/NOTICE | |
| boardCn | string | Y | board_cn | 빈값 불가 | |
| fixYn | string | N | fix_yn | Y/N | |
| newFiles | File[] | N | tb_cm_attach_file | 10MB, 최대 5개 | 새로 추가할 파일 |
| deleteFileIds | number[] | N | file_id | - | 삭제할 기존 파일 ID 목록 |

**Output**
| 필드 | 타입 | 설명 |
|------|------|------|
| success | boolean | 성공 여부 |
| message | string | "수정되었습니다" |

**처리 로직**
```
1. 게시글 존재 확인 (del_yn = 'N')
2. 권한 확인: 세션 user_id == reg_user_id
3. 권한 없으면 403 에러
4. tb_cm_board UPDATE (board_title_nm, board_cn, board_type_cd, fix_yn, mod_user_id, mod_dt)
5. 삭제 파일 처리: deleteFileIds에 해당하는 파일 del_yn = 'Y'
6. 새 파일 처리: 스토리지 저장 + tb_cm_attach_file INSERT
7. 총 첨부파일 수 검증: 기존(삭제 제외) + 신규 <= 5개
8. 성공 응답 + 상세 화면 이동
```

**SQL 개요**
```sql
-- 게시글 수정
UPDATE tb_cm_board SET
    board_title_nm = :boardTitleNm,
    board_cn = :boardCn,
    board_type_cd = :boardTypeCd,
    fix_yn = :fixYn,
    mod_user_id = :userId,
    mod_dt = NOW()
WHERE board_id = :boardId AND del_yn = 'N';

-- 기존 파일 삭제
UPDATE tb_cm_attach_file SET del_yn = 'Y'
WHERE file_id = ANY(:deleteFileIds) AND ref_type_cd = 'BOARD' AND ref_id = :boardId;

-- 새 파일 등록 (FN-00004와 동일)
```

**업무 규칙**
- 수정 화면 진입 시 기존 데이터를 폼에 바인딩
- 첨부파일: 기존 파일 유지 + 새 파일 추가 + 기존 파일 삭제 가능
- 총 첨부파일 5개 초과 시 "첨부파일은 최대 5개까지 가능합니다" 에러
- 취소 시 "수정을 취소하시겠습니까? 변경사항이 저장되지 않습니다." 확인
