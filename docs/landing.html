Problem

SI 개발, 왜 이렇게 비효율적일까요?

전통적인 SI 개발 방식이 가진 고질적인 문제들

📄
반복되는 문서 작업
요구사항 분석, 화면 설계, 기능 명세서 작성... 비슷한 작업을 수백 번 반복하며 시간을 낭비합니다.
🔍
설계 검토의 한계
사람이 직접 검토하면 누락되는 충돌 기능, 영향도 분석 오류, 일관성 문제가 발생합니다.
📊
진행 현황 파악 어려움
수백 개 기능의 개발 진행 상태를 한눈에 파악하기 어렵고, 병목 구간을 찾는 것도 어렵습니다.
Core Features

개발 사이클 전체를 하나로

요구사항부터 구현까지, 모든 것을 하나의 플랫폼에서 관리하세요

📋
계층 구조 관리
요구사항 → 화면 → 기능의 명확한 계층 구조로 프로젝트 전체를 체계적으로 관리합니다. 트리 뷰로 전체 구조를 한눈에 파악할 수 있습니다.
🤖
AI 설계 자동화
기능 명세 작성 후 AI에게 검토 요청 한 번으로 설계 오류 탐지, 충돌 기능 분석, 영향도 평가, 상세설계 생성까지 자동으로 처리됩니다.
🔄
10단계 상태 관리
설계중부터 구현완료까지 10단계 상태 워크플로우로 모든 기능의 진행 현황을 정밀하게 추적합니다. AI 처리 중에는 자동 상태 전이가 이루어집니다.
📡
폴링 기반 AI 연동
AI 서비스가 30초마다 폴링하여 대기 중인 작업을 가져가는 방식으로 방화벽 환경에서도 안정적으로 동작합니다. 완료 시 자동으로 결과가 반영됩니다.
📖
표준가이드 AI 검토
코딩 표준, 설계 패턴, 보안 가이드를 등록하고 AI가 준수 여부를 자동으로 검토합니다. 일관된 코드 품질을 유지하는 가장 스마트한 방법입니다.
⚡
실시간 현황 대시보드
전체 기능의 상태 분포, AI 작업 진행 현황, 최근 AI 활동 피드를 실시간으로 모니터링합니다. 30초마다 자동 갱신으로 항상 최신 상태를 유지합니다.
Workflow

AI와 함께하는 5단계 개발 사이클

처음 한 번만 설정하면, AI가 알아서 이어받아 진행합니다

1
요구사항 & 화면 등록
고객의 RFQ(입찰요청서)를 시스템에 등록하고 GS가 분석 내용을 작성합니다. 화면 유형(목록/상세/팝업/탭)과 레이아웃을 시각적으로 설계합니다.
요구사항 등록 화면 설계 레이아웃 에디터
2
기능 명세 작성
각 화면별 기능을 등록하고 마크다운으로 상세 명세를 작성합니다. 데이터 흐름(READ/WRITE 대상 테이블), 관련 파일, 참조 프로그램도 함께 기록합니다.
기능 명세 (Markdown) 데이터 흐름 참조 프로그램
3
AI 검토 요청 → 자동 처리
상태를 검토요청으로 변경하면 AI 작업이 자동 생성됩니다. AI가 설계 오류, 충돌 기능, 영향도를 분석하여 마크다운 형태의 검토 결과를 반환합니다.
AI 설계 검토 (REVIEW) 충돌 기능 탐지 영향도 분석
4
AI 상세설계 생성 → GS 컨펌
검토 결과를 확인 후 설계요청으로 변경합니다. AI가 상세 설계 문서를 자동 생성하면, GS가 내용을 검토하고 수정한 뒤 컨펌합니다.
AI 상세설계 (DESIGN) GS 수정/보완 컨펌 승인
5
AI 코드 구현 → 완료
구현요청으로 변경하면 AI가 실제 코드를 생성합니다. GitLab MR URL이 연동되고 구현 피드백, 이슈 파일 목록이 자동으로 기능에 캐싱됩니다.
AI 코드 구현 (IMPLEMENT) GitLab MR 연동 구현 완료
