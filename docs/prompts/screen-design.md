## 출력 형식 (반드시 준수)

### 1. 화면 개요

- **목적**: (이 화면이 무엇을 위한 화면인지 1~2문장)
- **URL 경로**: (예: `/users/list`, `/orders/[id]/detail`)
- **폴더/파일명**: (예: `src/app/users/list/page.tsx`)

### 2. 화면 레이아웃 (ASCII 박스)

영역(Area) 단위로 화면 전체 레이아웃을 표현한다.
각 박스는 영역 하나를 나타내며, 내부 상세 UI는 적지 않는다.

```
+======================================================+
| [헤더/타이틀 영역]                                    |
+======================================================+
| [검색 영역 (SEARCH)]                                  |
+------------------------------------------------------+
| [목록 영역 (GRID)]                                    |
|                                                      |
+------------------------------------------------------+
```

표기 규칙:
- `=` 구분선: 헤더/고정 영역
- `-` 구분선: 일반 영역
- 각 박스 안에 영역명과 유형 표기

### 3. 영역 목록

| 영역명 | 유형 | 역할 |
|--------|------|------|
| 검색 영역 | SEARCH | 조건 입력 후 목록 조회 트리거 |
| 목록 영역 | GRID | 조회 결과 테이블 표시 |

영역 유형: SEARCH | GRID | FORM | INFO_CARD | TAB | FULL_SCREEN

### 4. 영역 간 데이터 흐름

사용자 인터랙션에 따른 영역 간 연동 관계를 서술한다.

1. [검색 영역]에서 [조회] 클릭 → [목록 영역] 데이터 갱신
2. [목록 영역]에서 행 선택 → [상세 영역] 데이터 표시
3. (추가 흐름이 있으면 계속 기술)
