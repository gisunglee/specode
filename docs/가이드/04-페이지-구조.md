# 04. 페이지 구조

## 라우팅 규칙

### URL 패턴
```
/[엔티티복수]           → 목록 페이지
/[엔티티복수]/[id]      → 상세 페이지
/[엔티티복수]/new       → 신규 생성 (모달 방식 선호)
```

### 예시
```
/tasks              과업 목록
/requirements       요구사항 목록
/unit-works         단위업무 목록
/screens            화면 목록
/screens/[id]       화면 상세
/functions          기능 목록
/functions/[id]     기능 상세
```

---

## 목록 페이지 표준 구조

### 레이아웃 구성
```
+----------------------------------------------------------+
| [페이지 제목]                         [액션 버튼들]      |
+----------------------------------------------------------+
| [필터 영역: 검색 input, 드롭다운 필터들]                 |
+----------------------------------------------------------+
| [DataGrid: TanStack Table]                                |
|  컬럼1 | 컬럼2 | ... | 액션(수정/삭제)                  |
+----------------------------------------------------------+
| [페이지네이션]                                           |
+----------------------------------------------------------+
```

### 필수 구현 사항
- 목록은 `TanStack Table` 사용 (ColumnDef 배열 방식)
- 검색은 서버 사이드 (URL 쿼리 파라미터)
- 페이지당 기본 20건, pageSize 선택 가능 (20 / 50 / 100)
- 로딩 상태: 스켈레톤 또는 spinner
- 빈 목록: "데이터가 없습니다" 안내

## 상세 페이지 표준 구조

### 레이아웃 구성
```
+----------------------------------------------------------+
| ← 목록으로    [브레드크럼: 과업 > 요구사항 > 화면]       |
+----------------------------------------------------------+
| [기본 정보 카드]                      [액션 버튼들]      |
| 시스템 ID, 이름, 상태 등                                 |
+----------------------------------------------------------+
| [탭 영역]                                                |
| | 탭1 | 탭2 | 탭3 |                                     |
+----------------------------------------------------------+
| [탭 콘텐츠]                                              |
+----------------------------------------------------------+
| [하위 목록 (있는 경우)]                                  |
+----------------------------------------------------------+
```

### 탭 구성 예시
| 엔티티 | 탭 구성 |
|--------|---------|
| 요구사항 | 기본정보 / 상세명세 / 협의내용 / AI결과 |
| 화면 | 기본정보 / 영역목록 / 레이아웃 |
| 기능 | 기본정보 / 설계내용 / 구현가이드 |

## 공통 레이아웃

### 사이드바 + 본문 구조
```
+------------------+------------------------------------------+
| Sidebar          | Header (breadcrumb / user)               |
| - 로고           +------------------------------------------+
| - 메뉴 항목들    | Main Content                             |
|   (아이콘+텍스트)|                                          |
|                  |                                          |
+------------------+------------------------------------------+
```

### 사이드바 메뉴 계층
```
대시보드
기획 관리
  └ 과업 관리
  └ 요구사항 관리
  └ 단위업무 관리
화면 설계
  └ 화면 목록
  └ 영역 관리
개발 가이드
  └ 기능 목록
  └ AI 작업 현황
기타
  └ 표준 가이드
```

## 상태 관리 패턴

### 서버 상태
- TanStack Query `useQuery` — 목록/상세 조회
- TanStack Query `useMutation` — 생성/수정/삭제
- `queryClient.invalidateQueries()` — 뮤테이션 성공 후 캐시 무효화

### 로컬 UI 상태
- `useState` — 모달 open/close, 폼 값, 필터 값
- URL 쿼리 파라미터 — 검색어, 필터, 현재 페이지 (`useSearchParams`)

---

## 모달 패턴

### 생성/수정 모달
- Radix Dialog 기반
- 폼 검증은 클라이언트 단순 체크 (서버 검증 필수)
- 저장 버튼 클릭 → `useMutation` → 성공 시 모달 닫기 + 목록 갱신

### 삭제 확인 모달
- "정말 삭제하시겠습니까?" 간단 확인
- 삭제 불가 조건(하위 데이터 존재 등) → 409 응답 → 에러 메시지 표시
