# 02. 도메인 모델

## 엔티티 정의

### Task (과업)
RFP 원문의 대항목. 요구사항들의 상위 그룹.

| 필드 | 타입 | 설명 |
|------|------|------|
| taskId | int PK | |
| systemId | string | T-00001 형식 |
| name | string | 과업명 |
| category | string? | 분류 |
| content | text? | 세부내용 원문 |

---

### Requirement (요구사항)
기능 요구사항 단위. 과업에 속하며 화면의 상위.

| 필드 | 타입 | 설명 |
|------|------|------|
| requirementId | int PK | |
| systemId | string | RQ-00001 형식 |
| taskId | int FK? | 과업 (nullable) |
| name | string | 요구사항명 |
| originalContent | text? | RFP 원문 — 수정 금지 |
| currentContent | text? | 협의 후 최종본 |
| detailSpec | text? | 요구사항 명세서 (마크다운) |
| discussionMd | text? | 상세 협의 내용 — AI 참조용 |
| priority | string? | HIGH / MEDIUM / LOW |
| source | string | RFP / ADDED / CHANGED |

---

### UnitWork (단위업무)
화면 묶음. PRD 생성 단위. 요구사항 하위, 화면 상위.

| 필드 | 타입 | 설명 |
|------|------|------|
| unitWorkId | int PK | |
| systemId | string | UW-00001 형식 |
| requirementId | int FK | 요구사항 |
| name | string | 단위업무명 (예: 게시판 관리) |
| description | text? | 설명 |
| sortOrder | int | 정렬 순서 |

---

### Screen (화면)
단위업무에 속하는 개별 화면.

| 필드 | 타입 | 설명 |
|------|------|------|
| screenId | int PK | |
| systemId | string | SCR-00001 형식 |
| unitWorkId | int FK? | 단위업무 (nullable — 하위 호환) |
| requirementId | int FK? | 요구사항 (nullable) |
| name | string | 화면명 |
| screenType | string? | LIST / DETAIL / POPUP / TAB |
| spec | text? | 화면 명세 |
| layoutData | json? | 레이아웃 JSON |
| categoryL/M/S | string? | 대/중/소분류 |

---

### Area (영역)
화면 내 UI 구역 단위.

| 필드 | 타입 | 설명 |
|------|------|------|
| areaId | int PK | |
| areaCode | string | AR-00001 형식 |
| screenId | int FK | 화면 |
| name | string | 영역명 |
| areaType | string | SEARCH / GRID / FORM / INFO_CARD / TAB |
| status | string | 설계 상태 |
| aiFeedback | text? | AI 설계 검토 결과 |

---

### Function (기능)
영역 내 개별 기능. AI 처리의 최소 단위.

| 필드 | 타입 | 설명 |
|------|------|------|
| functionId | int PK | |
| systemId | string | FID-00001 형식 |
| areaId | int FK | 영역 |
| name | string | 기능명 |
| spec | text? | 기능 명세 |
| phase | string | DRAFT / REVIEW / DESIGN / IMPL |
| phaseStatus | string | IDLE / REQUESTED / PROCESSING / DONE |
| confirmed | bool | 컨펌 여부 |
| aiInspFeedback | text? | AI 검토 피드백 |
| aiDesignContent | text? | AI 상세 설계 |
| aiImplFeedback | text? | AI 구현 가이드 |

---

### AiTask (AI 태스크)
비동기 AI 처리 큐. AI 워커가 폴링해서 처리.

| 필드 | 타입 | 설명 |
|------|------|------|
| aiTaskId | int PK | |
| systemId | string | ATK-00001 형식 |
| refTableName | string | 대상 테이블명 |
| refPkId | int | 대상 PK |
| taskType | string | INSPECT / DESIGN / IMPLEMENT / PLANNING |
| taskStatus | string | NONE → RUNNING → SUCCESS/FAILED |
| spec | text | AI에게 전달할 입력 스펙 (JSON) |
| feedback | text? | AI 결과 저장 |

---

## AI 태스크 상태 흐름

```
NONE → [워커 start] → RUNNING → [워커 complete] → SUCCESS
                                                  → AUTO_FIXED
                                                  → NEEDS_CHECK
                                                  → WARNING
                                                  → FAILED
```

## systemId 접두사 규칙

| 엔티티 | 접두사 | 예시 |
|--------|--------|------|
| Task | T | T-00001 |
| Requirement | RQ | RQ-00001 |
| UnitWork | UW | UW-00001 |
| Screen | SCR | SCR-00001 |
| Area | AR | AR-00001 |
| Function | FID | FID-00001 |
| AiTask | ATK | ATK-00001 |
| StandardGuide | SG | SG-00001 |
