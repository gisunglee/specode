# 01. 프로젝트 개요

## 프로젝트 목적

SI 프로젝트의 기획 → 설계 → 개발 라이프사이클을 AI가 보조하는 웹 관리 시스템.
기획자가 요구사항을 등록하면 AI가 설계 검토 · 상세 설계 · 구현 가이드를 비동기로 처리한다.

## 핵심 사용자

| 역할 | 주요 업무 |
|------|----------|
| 기획자 (GS) | 요구사항 · 화면 · 영역 · 기능 등록, AI 결과 검토 |
| 개발자 | 기능 명세 확인, 구현 가이드 참조 |
| PM | 진행 현황 대시보드 모니터링 |

## 도메인 계층 (변경 불가 핵심 구조)

```
과업(Task)
  └─ 요구사항(Requirement)
       └─ 단위업무(UnitWork)          ← 화면 묶음 · PRD 전송 단위
            └─ 화면(Screen)
                 └─ 영역(Area)
                      └─ 기능(Function)
                           └─ AI태스크(AiTask)  ← 비동기 AI 처리 큐
```

## 프로젝트 목표 (SPECODE v1 교훈)

- AI 태스크는 **폴링 기반 비동기** 처리 (동기 호출 X)
- 모든 엔티티에 `systemId` 채번 (RQ-00001 형식) — 사람이 읽을 수 있는 식별자
- 콘텐츠 변경 이력은 별도 버전 테이블로 관리
- API 응답은 `{ success, data, pagination? }` 단일 형식 통일
