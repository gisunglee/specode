# 07. AI 연동

## 아키텍처 개요

```
[웹 앱] ──► tb_ai_task 등록 (NONE)
                  │
                  ▼
[AI 워커] ──► GET /api/ai/tasks  (폴링)
           ──► PATCH /api/ai/tasks/{id}/start   (RUNNING)
           ──► (AI 분석 처리)
           ──► POST /api/ai/tasks/{id}/complete (SUCCESS/FAILED)
                  │
                  ▼
[onTaskComplete 훅] ──► 대상 엔티티 자동 업데이트
```

**핵심 원칙**: AI 처리는 반드시 **비동기 폴링** 방식. 동기 호출 절대 금지.

---

## AiTask 엔티티

```prisma
model AiTask {
  aiTaskId    Int      @id @default(autoincrement()) @map("ai_task_id")
  systemId    String   @map("system_id")       // ATK-00001
  refTableName String  @map("ref_table_name")  // "tb_function"
  refPkId     Int      @map("ref_pk_id")       // 대상 PK
  taskType    String   @map("task_type")        // INSPECT / DESIGN / IMPLEMENT / PLANNING
  taskStatus  String   @default("NONE") @map("task_status")
  spec        String                             // AI에게 전달할 입력 (JSON or 마크다운)
  feedback    String?                            // AI 결과 저장
  comment     String?                            // 추가 요청사항 (사용자 입력)
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")

  @@map("tb_ai_task")
}
```

## 태스크 상태 흐름

```
NONE
  └─[AI 워커 start]─► RUNNING
                         ├─► SUCCESS      정상 완료
                         ├─► AUTO_FIXED   AI가 자동 수정한 경우
                         ├─► NEEDS_CHECK  사람 검토 필요
                         ├─► WARNING      경고 있으나 완료
                         └─► FAILED       처리 실패
```

---

## taskType 종류

| taskType | 대상 | 설명 |
|----------|------|------|
| INSPECT | Function / Area | 설계 검토 — 완성도·명확성·일관성 점검 |
| DESIGN | Function / Area | 상세 설계 — 데이터 흐름, 로직, API 설계 |
| IMPLEMENT | Function | 구현 가이드 — 코드 수준 구현 방법 |
| PLANNING | PlanningDraft | 기획 보드 처리 (IA, PROCESS, ERD, MOCKUP) |

## AiTask 등록 방법

```ts
// 웹 앱에서 AI 태스크 등록
const systemId = await generateSystemId("ATK");
await prisma.aiTask.create({
  data: {
    systemId,
    refTableName: "tb_function",
    refPkId: functionId,
    taskType: "INSPECT",
    taskStatus: "NONE",
    spec: JSON.stringify({ functionId, name, spec, areaName, screenName }),

  }
```

## onTaskComplete 훅

AI 워커가 완료 API를 호출하면 `onTaskComplete` 훅이 자동 실행된다.

### 파일 위치
```
src/app/api/ai/_lib/onTaskComplete.ts
```

### 처리 로직
```ts
// SUCCESS | AUTO_FIXED 상태일 때만 대상 엔티티 업데이트
if (status === "SUCCESS" || status === "AUTO_FIXED") {
  switch (refTableName) {
    case "tb_function":
      // taskType별로 적절한 필드 업데이트
      // INSPECT → aiInspFeedback, status → REVIEW_DONE
      // DESIGN  → aiDesignContent, status → DESIGN_DONE
      // IMPLEMENT → aiImplFeedback, status → IMPL_DONE
      break;
    case "tb_area":
      // aiFeedback 업데이트
      break;
    case "tb_planning_draft":
      // resultContent 업데이트
      break;
  }
  // 변경 이력 저장
  await saveContentVersion(tableName, pkId, fieldName, oldValue, "AI");
}
```

```
docs/prompts/{table_short}-{taskType_lower}.md
```

## AI 워커 API

### 대기 태스크 조회
```http
GET /api/ai/tasks?limit=10&taskType=INSPECT
X-API-Key: {AI_API_KEY}
```

### 태스크 시작
```http
PATCH /api/ai/tasks/{aiTaskId}/start
X-API-Key: {AI_API_KEY}
```

### 태스크 완료
```http
POST /api/ai/tasks/{aiTaskId}/complete
X-API-Key: {AI_API_KEY}
Content-Type: application/json

{
  "status": "SUCCESS",
  "feedback": "## 분석 결과\n\n..."
}
```

## 콘텐츠 버전 이력 저장

AI가 필드를 변경할 때는 반드시 이력을 남긴다.

```ts
import { saveContentVersion } from "@/lib/contentVersion";

await saveContentVersion(
  "tb_function",   // 테이블명
  functionId,      // PK
  "ai_design_content", // 필드명
  oldContent,      // 변경 전 값
  "AI"             // 변경자
);
```

## 주의사항

| 상황 | 주의 |
|------|------|
| 동기 AI 호출 | 절대 금지 — 반드시 비동기 폴링 |
| RUNNING 상태 | 동일 대상 중복 태스크 생성 방지 로직 필요 |
| FAILED 처리 | 대상 엔티티 상태 원복 필요 |
| spec 필드 | JSON 문자열로 저장, 파싱은 AI 워커 측에서 |
| 한글 전송 | curl 대신 Python urllib 사용 (UTF-8 보장) |
