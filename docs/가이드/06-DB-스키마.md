## Prisma 스키마 규칙

### 파일 위치
```
prisma/
  schema.prisma   # 스키마 정의
  seed.ts         # 초기 데이터 (시퀀스 테이블 포함)
  migrations/     # 마이그레이션 이력
```

### 네이밍 규칙
```prisma
model SomeEntity {
  someEntityId  Int      @id @default(autoincrement()) @map("some_entity_id")
  systemId      String   @map("system_id")
  createdAt     DateTime @default(now()) @map("created_at")
  updatedAt     DateTime @updatedAt @map("updated_at")

  @@map("tb_some_entity")
}
```

| 규칙 | 설명 |
|------|------|
| 모델명 | PascalCase (예: `UnitWork`) |
| 필드명 | camelCase (예: `unitWorkId`) |
| 테이블명 | `tb_` 접두사 + snake_case |
| 컬럼명 | snake_case (`@map` 사용) |
| PK | `@id @default(autoincrement())` |

## 공통 필드 패턴

### 모든 엔티티 공통
```prisma
systemId  String   @map("system_id")    // 사람이 읽는 ID (RQ-00001)
useYn     String   @default("Y") @map("use_yn")  // 논리 삭제
createdBy String?  @map("created_by")
createdAt DateTime @default(now()) @map("created_at")
updatedAt DateTime @updatedAt @map("updated_at")
```

### AI 결과 저장 필드
```prisma
aiFeedback     String?  @map("ai_feedback")      // AI 검토 피드백
aiDesignContent String? @map("ai_design_content") // AI 상세 설계
aiImplFeedback String?  @map("ai_impl_feedback")  // AI 구현 가이드
```

## 시퀀스 테이블

모든 `systemId`는 `tb_sequence` 테이블에서 채번한다.

```prisma
model Sequence {
  prefix    String @id
  lastValue Int    @default(0) @map("last_value")

  @@map("tb_sequence")
}
```

### seed.ts에 초기 등록
```ts
const prefixes = ["T", "RQ", "UW", "SCR", "AR", "FID", "ATK", "SG"];
for (const prefix of prefixes) {
  await prisma.sequence.upsert({
    where: { prefix },
    update: {},
    create: { prefix, lastValue: 0 }
  });
}
```

## 콘텐츠 버전 이력 테이블

AI가 변경하는 필드는 변경 이력을 남긴다.

```prisma
model ContentVersion {
  versionId   Int      @id @default(autoincrement()) @map("version_id")
  tableName   String   @map("table_name")   // "tb_function"
  pkId        Int      @map("pk_id")
  fieldName   String   @map("field_name")   // "ai_design_content"
  content     String                         // 변경 전 내용
  changedBy   String?  @map("changed_by")   // "AI" or 사용자명
  createdAt   DateTime @default(now()) @map("created_at")

  @@map("tb_content_version")
}
```

## 마이그레이션 규칙

```bash
# 스키마 변경 후
npx prisma migrate dev --name "add_unit_work"

# 클라이언트 재생성
npx prisma generate

# DB 초기화 (개발 환경만)
npx prisma migrate reset

# Prisma Studio (DB GUI)
npx prisma studio
```

### 마이그레이션 이름 규칙
```
add_{entity}          신규 엔티티 추가
add_{field}_to_{table}  기존 테이블 필드 추가
alter_{table}_{action}  기존 테이블 구조 변경
```

## 관계 정의 패턴

### 1:N 관계
```prisma
model Parent {
  parentId Int     @id @default(autoincrement()) @map("parent_id")
  children Child[]
  @@map("tb_parent")
}

model Child {
  childId  Int    @id @default(autoincrement()) @map("child_id")
  parentId Int    @map("parent_id")
  parent   Parent @relation(fields: [parentId], references: [parentId])
  @@map("tb_child")
}
```

### nullable FK (하위 호환)
```prisma
// FK를 nullable로 선언하면 기존 데이터 영향 없이 추가 가능
parentId Int? @map("parent_id")
parent   Parent? @relation(fields: [parentId], references: [parentId])
```

## Prisma 클라이언트 싱글톤

```ts
// src/lib/prisma.ts
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

export const prisma =
  globalForPrisma.prisma ??
  new PrismaClient({ log: ["error"] });

if (process.env.NODE_ENV !== "production") {
  globalForPrisma.prisma = prisma;
}
```

## 주의사항

| 상황 | 주의 |
|------|------|
| Prisma 7.x | 사용 금지 — datasource 포맷 변경으로 호환 안 됨 |
| 논리 삭제 | 조회 시 `where: { useYn: "Y" }` 반드시 포함 |
| JSON 필드 | SQLite에서는 Text 저장, PostgreSQL에서는 jsonb |
| 트랜잭션 | 채번+생성은 반드시 `prisma.$transaction` 사용 |
