# SPECODE — AI Dev Hub 시스템 개요

> 이 문서는 시스템의 목적, 핵심 개념, 화면 구성을 요약합니다.
> AI가 목업(UI Mockup)을 생성하거나 시스템을 재설계하는 데 활용할 수 있습니다.

---

## 1. 시스템 목적

**SPECODE(AI Dev Hub)**는 SI(시스템 통합) 개발 프로젝트에서 **비즈니스 분석가(GS)**와 **AI**가 협력하여 요구사항 분석 → 화면 설계 → 기능 명세 → 코드 구현까지 이어지는 개발 사이클을 자동화하는 **AI 주도 개발 관리 시스템**입니다.

### 핵심 가치

| 가치 | 설명 |
|------|------|
| **단계적 AI 협업** | 기능 명세 작성 → AI 검토 → AI 설계 → AI 코드 구현 단계별 자동화 |
| **진행 추적** | 기능별 상태를 10단계로 추적하여 개발 현황을 한눈에 파악 |
| **표준 준수** | 표준가이드를 AI가 검토하여 코딩 표준 일관성 유지 |
| **구조적 관리** | 요구사항 → 화면 → 기능의 계층 구조로 체계적인 관리 |

## 2. 핵심 개념 (Core Concepts)

### 2.1 계층 구조

```
요구사항 (Requirement)
    └── 화면 (Screen)
            └── 기능 (Function)
                    └── AI 작업 (AiTask)
```

- **요구사항**: 고객의 RFQ(입찰요청서)를 등록하고 GS가 분석한 내용을 기록
- **화면**: 시스템에서 사용자가 보는 UI 화면 단위 (목록화면, 상세화면, 팝업 등)
- **기능**: 화면 안에서 실제로 동작하는 개별 기능 단위 (검색, 저장, 삭제 등)
- **AI 작업**: AI에게 요청하는 작업 (검토, 설계, 구현, 영향도 분석 등)

### 2.2 기능 상태 흐름

```
DRAFT (설계중)
    ↓ GS가 기능 명세 작성 완료
REVIEW_REQ (검토요청) ──→ AI_REVIEWING (AI검토중)
    ↓ AI 검토 완료
REVIEW_DONE (검토완료)
    ↓ GS 확인 후 설계 요청
DESIGN_REQ (설계요청) ──→ AI_REVIEWING
    ↓ AI 설계 완료
DESIGN_DONE (설계완료)
    ↓ GS 컨펌
CONFIRM_Y (컨펌)
    ↓ GS 구현 요청
IMPL_REQ (구현요청) ──→ AI_IMPLEMENTING (AI구현중)
    ↓ AI 구현 완료
IMPL_DONE (구현완료)
```

### 2.3 AI 연동 방식

AI 서비스는 **폴링(Polling)** 방식으로 연동됩니다:
1. GS가 기능 상태를 REVIEW_REQ/DESIGN_REQ/IMPL_REQ로 변경 → AI 작업(AiTask) 자동 생성
2. AI 서비스가 30초마다 `GET /api/ai/tasks`를 호출하여 대기 중인 작업을 가져감
3. AI가 처리 완료 후 `POST /api/ai/tasks/{id}/complete`로 결과를 전달
4. 시스템이 기능의 AI 피드백 필드를 업데이트하고 상태를 자동 전이

## 3. 화면 목록 및 UI 설명

### 3.1 대시보드 (/)

**목적**: 프로젝트 전체 진행 현황을 한눈에 파악

```
┌──────────────────────────────────────────────────────────────┐
│  헤더: "AI Dev Hub"                                           │
├──────────────────────────────────────────────────────────────┤
│  상태 카드 5개 (가로 배열):                                    │
│  [설계중 N] [검토요청 N] [검토완료 N] [AI구현중 N] [구현완료 N] │
├──────────────────────────────────────────────────────────────┤
│  하이라이트 카드 2개:                                          │
│  [컨펌 대기 N개]  [AI 작업 중 N개]                            │
├──────────────────────────────────────────────────────────────┤
│  최근 AI 활동 (그리드, 2열):                                   │
│  ┌────────────────┐  ┌────────────────┐                      │
│  │ [설계검토] 성공  │  │ [코드구현] 진행중│                    │
│  │ FID-00001 기능명│  │ FID-00002 기능명│                    │
│  │ 피드백 미리보기  │  │ 피드백 미리보기  │                   │
│  │ 요청: 10분 전   │  │ 요청: 2분 전   │                     │
│  └────────────────┘  └────────────────┘                      │
└──────────────────────────────────────────────────────────────┘
```

- 상태 카드 클릭 → 해당 상태로 필터된 기능 목록으로 이동
- AI 활동 카드 클릭 → 기능 상세 페이지로 이동
- 30초마다 자동 새로고침

### 3.7 표준가이드 (/standard-guides)

**목적**: AI가 준수해야 할 코딩/설계 표준 문서 관리 및 AI 검토 요청

```

│GD-0002 │ DATA     │페이징 처리  │          │ 활성     │🔍✏🗑│
│GD-0003 │ AUTH     │JWT 인증     │ 🔄검토요청│ 비활성   │🔍✏🗑│
└──────────────────────────────────────────────────────────┘
```

- **활성여부**: 클릭하면 즉시 토글 (다이얼로그 없이)
- **상태**: 클릭하면 AI 검토 이력 다이얼로그 열림
- **🔍**: AI 검토 요청 (INSPECT 작업 생성)

## 6. 주요 UX 패턴

1. **스티키 탭 네비게이션**: 긴 페이지에서 스크롤해도 탭이 상단에 고정됨
2. **상태 뱃지**: 모든 목록에서 컬러 코딩된 상태 표시
3. **AI 요청 확인 다이얼로그**: AI에게 보낼 코멘트를 추가로 입력 가능
4. **자동 새로고침**: AI 현황(10초), 대시보드(30초), 표준가이드(8초)
5. **배치 작업**: 체크박스 선택 후 일괄 상태 변경 가능
6. **마크다운 에디터**: 작성(edit)/미리보기(preview) 전환 가능
7. **ID 자동 생성**: 저장 시 RQ-00001 형식의 ID 자동 부여
