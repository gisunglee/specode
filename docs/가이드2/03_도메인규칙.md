# 도메인 규칙

> ✅ SPECODE 비즈니스 로직 기준. 새 구현에서도 동일하게 적용.
> 이 파일의 규칙을 어기는 코드는 잘못된 코드임.

---

## 시스템 계층 구조

```
프로젝트(Project)
  └─ 과업(Task)
       └─ 요구사항(Requirement)
            ├─ 사용자스토리(UserStory)
            │    └─ 인수기준(AcceptanceCriteria)
            └─ 단위업무(UnitWork)
                 └─ 화면(Screen)
                      └─ 영역(Area)
                           └─ 기능(Function)
                                └─ AI태스크(AiTask)
```

상위 항목 삭제 시 하위 항목 전체 연쇄 논리 삭제.

## systemId 채번 규칙

모든 도메인 엔티티는 사람이 읽을 수 있는 ID를 가진다.
DB auto-increment PK와 별도로 `tb_sequence` 테이블 기반 채번.

| prefix | 대상 | 예시 |
|--------|------|------|
| T | 과업 | T-00001 |
| RQ | 요구사항 | RQ-00001 |
| US | 사용자스토리 | US-00001 |
| UW | 단위업무 | UW-00001 |
| SCR | 화면 | SCR-00001 |
| AR | 영역 | AR-00001 |
| FID | 기능 | FID-00001 |
| ATK | AI태스크 | ATK-00001 |
| SG | 표준가이드 | SG-00001 |
| PRJ | 프로젝트 | PRJ-00001 |

채번 함수: `generateSystemId(prefix)` — 동시성 고려한 트랜잭션 처리 필수.

## 삭제 정책

**물리 삭제 없음.** 모든 삭제는 논리 삭제.

```
delYn = 'N'  →  정상
delYn = 'Y'  →  삭제됨
```

조회 시 항상 `WHERE delYn = 'N'` 조건 포함. 빠뜨리면 버그.

예외: AI 태스크 첨부파일(임시 파일)만 물리 삭제 허용.

## 기능(Function) 상태 흐름

```
DRAFT
  │ 검토 요청
  ▼
IN_REVIEW ──────────────── AI 태스크 자동 생성
  │ AI 처리 완료 (자동)
  ▼
FEEDBACK_DONE
  │ 설계자 확인
  ▼
REVIEWING
  │ 담당자만 가능
  ▼
CONFIRMED
  │ 개발 시작
  ▼
IN_PROGRESS
  │ 개발 완료
  ▼
DONE
```

**롤백 규칙**: DONE을 제외한 모든 상태에서 이전 상태로 되돌리기 가능.
**CONFIRMED 전환**: 해당 기능의 담당자(assignee) 또는 OWNER/ADMIN만 가능.

## AI 태스크 상태 흐름

```
NONE → RUNNING → SUCCESS
                → AUTO_FIXED
                → NEEDS_CHECK
                → WARNING
                → FAILED
```

- `NONE`: 대기 중 (폴링 대상)
- `RUNNING`: 처리 중 (Claude Code가 start API 호출 후)
- `SUCCESS / AUTO_FIXED`: 성공 → onTaskComplete 훅 자동 실행
- `NEEDS_CHECK / WARNING`: 반쪽 성공 → 사람 확인 필요
- `FAILED`: 실패 → 재처리 버튼 노출

## AI 태스크 유형별 동작

| taskType | 트리거 | 결과 저장 위치 |
|----------|--------|---------------|
| INSPECT | 기능 IN_REVIEW + INSPECT 선택 | Function.aiInspFeedback |
| DESIGN | 기능 IN_REVIEW + DESIGN 선택 | Function.aiDesignContent |
| IMPLEMENT | 기능 IN_REVIEW + IMPLEMENT 선택 | Function.aiImplFeedback |
| MOCKUP | 영역 이미지 업로드 후 요청 | Area.mockupContent |
| PLANNING | 기획 보드 AI 생성 버튼 | PlanningDraft.resultContent |

AI 태스크 spec에는 **단독 명세가 아닌 상위 계층 전체 컨텍스트** 포함 필수.
(기능만 보내면 AI 품질 크게 저하)

## 프로젝트 복사 규칙

복사 대상: 과업·요구사항·사용자스토리·단위업무·화면·영역·기능·AI결과물
복사 제외: 멤버 목록, AI 태스크 이력
복사본 생성자가 자동으로 OWNER.

---

## OWNER 탈퇴 처리

- 팀원 있음: 다른 멤버에게 OWNER 양도 후 탈퇴
- 혼자 + 데이터 있음: 확인 3회 → 프로젝트 전체 삭제
- 혼자 + 데이터 없음: 즉시 삭제

## 순서(sortOrder) 관리

목록이 있는 모든 엔티티는 `sortOrder` 필드 보유.
드래그앤드롭으로 순서 변경 시 즉시 저장.
신규 생성 시 현재 max(sortOrder) + 1.

---

## 이력(History) 자동 적재 대상

명세 스펙 필드 저장 시 이전 값 자동 보존.
항목명/상태 변경은 이력 미적재 (명세 내용 변경만 대상).
이력은 읽기 전용 — 삭제 불가.
